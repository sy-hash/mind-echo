セッション引き継ぎノートを作成してください。

## ルール

- 保存先: `.claude/handovers/` ディレクトリ（存在しなければ作成）
- ファイル名: `YYYY-MM-DD_HHmm.md` 形式（現在時刻ベース）
- 同名ファイルが存在する場合は末尾に `_2`, `_3` と連番を付与
- 簡潔に箇条書き中心で記述
- 事実ベースで書き、推測や曖昧な表現を避ける

## テンプレート

以下のセクション構成で引き継ぎノートを生成してください：

```markdown
# セッション引き継ぎノート YYYY-MM-DD HH:mm

## ブランチ
- 現在のブランチ名と状態（未コミットの変更有無など）

## 今回やったこと
- 作業内容と進捗を箇条書きで記載

## 決定事項
- 確定した設計判断や方針

## 捨てた選択肢と理由
- 検討したが不採用にしたアプローチとその理由

## ハマりどころ
- 遭遇したエラーや想定外の挙動と対処法

## 学び
- 今回のセッションで得られた知見

## 次にやること
- [ ] 未完了タスク（優先度順）

## 関連ファイル
- 変更・追加した主要ファイルパス一覧
```

## 手順

1. `git status` と `git diff --stat` で現在の作業状態を確認
2. 今回のセッションでの会話内容を振り返り、各セクションを埋める
3. 該当しないセクションは「特になし」と記載（セクション自体は残す）
4. ファイルを保存したらパスをユーザーに伝える
